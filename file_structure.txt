--------------------------------------------------------------------------------
2. Detailed File Analysis
--------------------------------------------------------------------------------

----------------------------------------
[A] index.html
----------------------------------------
* Function:
  Provides the structured visual layout of the single-page application. Contains the
  application container, the navigation sidebar, modal definitions (e.g., terms of
  service, about, contact, pet setup), page views (containers styled with the
  ".page-view" class), and the floating 3D avatar element.

* Libraries & External Resources:
  - Google Fonts (Nunito: weights 400, 600, 700, 800, 900)
  - Native Emojis (used for all card, layout, and button icons to avoid weightier
    external icon font libraries)

* Core Concepts:
  - Semantic HTML5 (header, main, aside, footer, section tags).
  - Single Page Application (SPA) container structuring. Page views are defined in the
    DOM and toggled dynamically.
  - Dialog and overlay styling via CSS positioning.

----------------------------------------
[B] styles.css
----------------------------------------
* Function:
  Manages the entire presentation layer, colors, layouts, responsiveness, transitions,
  and customized scrolling controls.

* Core Concepts:
  - CSS Variables & Theming: Declares standard variables under :root and overrides them
    under [data-theme="midnight"] and [data-theme="forest"] to support on-the-fly pastel
    color changes.
  - Viewport-Resilient Grids: Uses CSS Grid repeat(auto-fill, minmax(min(100%, X), 1fr))
    for responsive layout wrapping on viewports down to 320px without overflow.
  - Safe-Area Insets: Integrates env(safe-area-inset-...) rules to position components
    safely around notches, camera cuts, and software navigation bars.
  - Media Queries for Pointer Capability: Scopes hover scaling transformations inside
    @media (hover: hover) blocks, preventing stick-and-zoom scroll issues on mobile screens.
  - Firefox Scrollbar Standards: Implements scrollbar-width and scrollbar-color variables
    so Firefox matches the custom thin WebKit scrollbar styling.
  - Keyframe Animations: Custom animations for dropdown items (@keyframes slideDownMenu)
    and entrance animations (@keyframes modalFlatEntrance).

----------------------------------------
[C] app.js
----------------------------------------
* Function:
  Controls the business logic, state data, routing actions, and event processing.

* Libraries & API Implementations:
  - LocalStorage API: Saves data as serialized JSON under the key "pawtrack_data".
  - Web Audio API: Programmatic synthesizer that plays custom audio patterns (meows,
    barks, chirps) using oscillator and gain nodes.
  - Notifications API: Request permissions and triggers native OS reminders.
  - History Hash API: Listens to window.addEventListener('hashchange') for client-side
    single-page routing.

* Core Concepts:
  - Data Migration: Checks for legacy single-pet formats and auto-converts them to the
    multi-pet schema on startup.
  - Event Delegation: Attaches listeners to elements dynamically during updates.
  - State Tracking and Math Helpers: Includes birthdate converters to calculate raw age
    and species-adjusted human equivalents.
  - SVG Rendering: Generates gridlines, chart-lines, and interactive points for the weight
    log widget dynamically using document.createElementNS.

----------------------------------------
[D] vite.config.js
----------------------------------------
* Function:
  Vite configuration file.

* Core Concepts:
  - Dev Server Customization: Configures the developer server to listen on port 3000 and
    automatically open the browser when the dev script is invoked.

----------------------------------------
[E] package.json
----------------------------------------
* Function:
  Node.js manifest detailing scripts and project dependency metrics.

* Dependencies:
  - vite (v5.4.21): Configured as a development dependency.

* Scripts:
  - dev: "vite" (starts local development server)
  - build: "vite build" (compiles codebase into optimized assets under dist/)
  - preview: "vite preview" (runs local web server to preview built files)

----------------------------------------
[F] public/
----------------------------------------
* Function:
  Vite static asset directory. All files placed in this folder are mapped directly
  to the root of the server during development and copied to the build folder during
  production compiling.
